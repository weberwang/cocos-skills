## 编排动作

1. 先验收代码代理和资源代理的生产结果：
   - `baseline_revision`、冻结视觉版本及内容哈希必须与当前工作流一致。
   - `changed_paths` 必须完全位于各自 `allowed_paths` 内，且路径所有权不得重叠。
   - 工件、验收检查和证据必须完整，P0 检查全部通过。
   - 仅有“已完成”声明时拒绝验收。

2. 两项 Cocos Editor 写入不得并行。即使文件路径不重叠，同一时刻也只能有一个 `active_cocos_writers`。

3. 将编辑器写入拆成两个串行批次：
   - 批次一：授予场景节点任务唯一写入权。先读取当前场景和稳定节点标识，确认场景状态及任务所有权；随后创建节点、添加所需组件并保存场景。
   - 批次一完成后：重新读取场景，验证节点层级、节点标识、组件类型及关键属性，保存读回证据。验证通过后才能释放唯一写入权。
   - 批次二：再授予资源导入与绑定任务唯一写入权。先读取目标节点和现有资源状态；随后导入图片，等待导入完成，读取资源元数据，并把对应 SpriteFrame 绑定到目标 Sprite 组件。
   - 批次二完成后：重新读取资源和场景，验证资源导入状态、资源 UUID、SpriteFrame 引用、目标节点及组件属性，并保存场景。最终再次读回确认保存结果。

4. 任一批次读回失败，立即停止后续依赖写入，不得让另一个代理继续操作。为失败项创建限定路径的修复任务，通过原验收检查后再恢复集成。

## 状态迁移

```text
production/running
→ production/passed
→ integration/pending
→ integration/running
→ integration/passed
```

每次迁移必须记录完整的 canonical 七字段，并以非空证据支持。

异常分支：

```text
integration/running → integration/failed
integration/failed → integration/retrying → integration/running
integration/running → integration/blocked
```

出现以下情况时进入 `blocked`：

- 无法取得唯一 Cocos Editor 写入权。
- 本次 MCP 能力快照缺少所需的读取、资源导入、节点创建、组件设置、场景保存或读回能力。
- 任务路径所有权冲突。
- 缺少必需输入、冻结版本或内容哈希。
- 前一批次没有通过读回验证。

发现两个代理并发写入同一个 Cocos Editor 时，属于严重失败：立即停止全部写入，保留现场证据并请求人工处置，不自动重试。

## 门禁

- 路径门禁：两个任务的 `allowed_paths` 必须明确且无重叠；越界写入直接拒绝验收。
- 单写者门禁：`active_cocos_writers` 最多一个，第二个任务只能排队。
- 能力门禁：只能使用本次运行能力快照中存在、参数 schema 明确且位于允许列表内的操作。
- 安全门禁：禁止删除、移动、任意脚本执行、编辑器重启或退出等默认拒绝操作。
- 覆盖门禁：若任何操作需要 `overwrite: true`，必须先取得针对具体目标的人工批准并记录证据。
- 批次门禁：每个写入批次必须完成“先查询、再修改、再读取验证”，通过后才可释放写入权或启动下一批。
- P0/P1 门禁：P0 失败不可豁免；P1 只有绑定具体检查和工件哈希的有效人工豁免才能继续。
- 集成阶段本身没有常规人工批准门禁，但上述安全、能力和覆盖门禁仍必须满足。

## 证据要求

- 两个生产结果文件及其 `baseline_revision`、冻结输入版本和内容哈希。
- 各任务的 `allowed_paths`、实际 `changed_paths` 和路径所有权检查结果。
- 单写者锁的取得、释放及两个批次没有重叠的时间记录。
- 批次一写入前后的场景读回：
  - 场景路径与稳定标识。
  - 新建节点层级、节点标识、组件和关键属性。
  - 场景保存成功记录。
- 批次二写入前后的资源与场景读回：
  - 图片路径、导入状态、资源 UUID 和 SpriteFrame 元数据。
  - 目标节点、Sprite 组件及最终 SpriteFrame 引用。
  - 最终场景保存和再次读回结果。
- 每项验收检查的方法、结果和摘要。
- 非空日志、读回报告或截图证据路径。
- 最终集成工件的版本与内容哈希，以及供验证阶段使用的交接说明。
