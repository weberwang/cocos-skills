# MCP 安全策略

## 启动与能力快照

启动时先调用 `/health`，再调用 `tools/list` 或 `/capabilities`。将实际返回的工具名、参数 schema、版本、只读/写入分类、调用时间和服务健康状态保存到 `.cocos-workflow/reports/mcp-capabilities.json`。未出现在快照中的工具或参数一律不可使用。

运行时 `tools/list` 或 `/capabilities` 的实际结果是工具名称、参数和 schema 的唯一事实来源。静态文档、预设工具名、本文示例或既有能力快照与本次运行结果冲突时，以本次运行结果为准；需要的工具或参数缺失时，立即停止对应动作并将任务标记为 `blocked`，不得猜测别名、参数、默认值或替代工具。

## 工具允许列表

只允许使用本次运行能力快照中与下列操作对应的工具及其已声明参数；名称不同的服务必须由能力结果提供明确映射，不能按相似名称猜测：

| 类别 | 允许操作/工具 |
| --- | --- |
| 只读 | `/health`、`tools/list`、`/capabilities`、项目信息读取、资源查询、场景/节点/组件读取、控制台与构建状态读取 |
| 资源导入 | 资源导入、导入状态查询、资源元数据读取；仅写入任务 `allowed_paths` |
| 场景集成 | 创建节点、添加组件、设置允许属性、保存当前场景；仅由唯一 Cocos Editor 写入者执行 |
| 调试 | 启停 Cocos Editor Preview、读取/清理本次任务日志、非脚本式检查；本工作流的验证阶段不调用任何调试工具，游戏运行与判定由人工完成 |
| 构建 | 读取构建配置、启动 Web Mobile 构建、查询构建状态、读取构建日志和产物清单 |

默认拒绝列表优先于允许列表。默认禁止：

- `node_delete_node`
- `project_delete_asset`
- `project_move_asset`
- `project_save_asset`
- `debug_execute_script`
- 任意偏好设置写入
- 编辑器重启或退出

未知工具、任意代码执行、目录外写入和未声明的破坏性操作也默认禁止。

## 写入协议

1. 读取目标及其稳定标识，确认当前状态和任务所有权。
2. 校验工具在能力快照和允许列表中，并确认没有命中禁止列表。
3. 执行最小修改；任何 `overwrite: true` 必须先取得针对具体目标的人工批准并记录证据。
4. 重新读取目标，验证属性、引用、资源状态或构建状态与预期一致。
5. 保存读回证据；验证失败时停止后续依赖写入。

始终执行“先查询、再修改、再读取验证”。复杂预制体实例化失败时，不反复尝试不透明操作；改用按确定顺序创建节点、添加组件、设置属性和绑定资源的确定性节点重建，并在每个批次后读回验证。
